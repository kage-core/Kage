# Kage distribution plan

Goal: durable popularity = a genuinely useful tool × the right people seeing it ×
showing up to engage. No manipulation (no bought stars, vote rings, karma farming
— it gets detected and backfires). Everything below is honest reach + presence.

Positioning (settled): **shared, code-grounded memory for developers and their
coding agents — so a team stops rediscovering its own codebase.** Memory-first;
trust/grounding is *how it stays accurate*, not the identity; the benchmark is a
proof point, not the product.

## The 30-second "aha"
Lead every channel with the demo GIF (hallucinated→rejected, stale→withheld,
grounded→recalled) and the live viewer. People share what they instantly get.

## Flywheel
First ~50 genuine stars (network + launch day) → trending/listicles → more stars.
README keeps the GIF up top. Ship a short follow-up post cadence (build-in-public,
a "how grounding works" deep-dive) so it's not a one-spike.
