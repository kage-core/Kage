# Agent-surface capabilities: what Kage actually certifies

Kage attaches to coding agents in two ways: it **captures** what the agent does
(prompts, tool calls, results) and it **injects** repository memory into the
agent's context. Different surfaces support these to very different degrees. This
page is the honest matrix. The rule that governs everything below:

> A surface is labeled **automatic** only after a transcript-based smoke test
> proves it. Installed configuration alone never earns the label, and a captured
> telemetry stream (which proves capture) never implies injection.

The certification logic lives in
[`mcp/vnext/adapters/capability-matrix.ts`](../../mcp/vnext/adapters/capability-matrix.ts)
(`certifySurface`). The three-surface release gate lives in `mcp/kernel.ts`
(`agentSurfaceCertificationGate`).

## Capability levels

**Capture** (`CaptureLevel`): `automatic` · `manual` · `unavailable`.

**Injection** (`InjectionLevel`):

| Level | Meaning |
| --- | --- |
| `automatic_task` | Kage injects per task without the agent asking. |
| `automatic_session` | Kage injects session-level context via a native/session hook. |
| `gateway` | Kage injects at the measured proxy gateway on the wire. |
| `mcp_fallback` | The agent must call an MCP tool (`kage_context` / `kage_retrieve`) to pull context. Not automatic. |
| `unavailable` | No certified injection path. |

`counts_as_automatic_attachment` is `true` only when **capture is `automatic`**
and **injection is one of `automatic_task` / `automatic_session` / `gateway`**.
`mcp_fallback` and `unavailable` never count.

## How a surface earns "automatic injection"

`certifySurface` takes a smoke-test fixture: the number of capture events, the
sentinel Kage asked the surface to inject, the transcript the surface actually
produced, and a health status. Injection is certified automatic **only when the
sentinel string appears in the transcript** — and never above the surface's
declared *ceiling*. A surface whose ceiling is `mcp_fallback` can never be raised
by any transcript.

## The matrix

| Surface | Capture | Injection ceiling | Counts as automatic? | Notes |
| --- | --- | --- | --- | --- |
| `claude-code` | automatic | `automatic_session` | yes (when sentinel proven) | Native Claude Code hooks. |
| `anthropic-proxy` | automatic | `gateway` | yes (when sentinel forwarded) | Measured Phase A/D gateway. |
| `cursor` | automatic | `automatic_session` | yes (only on a certified version) | See Cursor section. |
| `codex` | automatic | `mcp_fallback` | **no** | See Codex section. |
| `other` | manual | `unavailable` | no | Unknown surfaces. |

## Cursor

Kage installs a project-level
[`.cursor/hooks.json`](../../plugin/cursor/hooks.json) (generated by
`cursorHooksConfig()` in
[`mcp/vnext/adapters/cursor-hooks.ts`](../../mcp/vnext/adapters/cursor-hooks.ts))
with `sessionStart`, `beforeSubmitPrompt`, `afterFileEdit`, and `stop` hooks. See
Cursor's documented hook configuration:
<https://cursor.com/blog/agent-best-practices>.

- `sessionStart` **may** return session-level context — but only counts as
  `automatic_session` after the certification smoke test proves the configured
  Cursor version actually delivers it.
- `beforeSubmitPrompt` records the task event only. Cursor's currently documented
  hook output does not reliably support returning per-prompt context, so Kage
  does **not** claim prompt-specific injection there.

If Cursor certification fails, the release gate stays failed. Kage does not
relabel the surface to keep the gate green.

## Codex

Kage consumes Codex OpenTelemetry events (prompts, approvals, tool results, MCP
usage, proxy decisions) for **automatic capture** via
[`plugin/codex/otel-config.toml`](../../plugin/codex/otel-config.toml) (generated
by `codexOtelConfig()` in
[`mcp/vnext/adapters/codex-otel.ts`](../../mcp/vnext/adapters/codex-otel.ts)).
OpenAI documents these events at
<https://openai.com/index/running-codex-safely/>.

Codex plugins package skills/apps/MCP but **do not** establish a general
Kage-owned session injection hook
(<https://help.openai.com/en/articles/20001256-plugins-in-codex/>). Therefore
Codex is reported as **automatic capture plus MCP fallback** for injection, and
`counts_as_automatic_attachment` is `false`. This holds even if a sentinel
appears in a transcript, because Codex's injection ceiling is `mcp_fallback`. The
label upgrades only when a real transcript-based injection certification (a
Kage-owned session hook) passes.

## The release gate

`agentSurfaceCertificationGate` requires successful automatic-attachment
certification for:

1. Claude Code native hooks (`automatic_session`).
2. A proxy-compatible agent using the measured gateway (`gateway`).
3. Cursor session-start injection on a certified version (`automatic_session`).

Codex is visible in the matrix but is **not** required to count as an automatic
attachment while it remains MCP fallback. Its presence never fails the gate, and
its label is never flipped to automatic by installed config alone.
