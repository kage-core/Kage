Latest package notes

v1.1.35

Current npm package for local-first repo memory, code graph, recall, viewer, PR checks, repo-intelligence maps, optional post-commit automation, mixed-language route indexing, confidence-scored call edges, workspace topic contracts, cross-repo co-change links, dashboard-first viewer sections, graph-first drawer workspace, floating graph toolbar, viewer Workspace Map drill-down, source graph Path Finder, and conservative cleanup review input.

  • The viewer now opens with a repo dashboard for readiness, memory coverage, graph health, risk, review, and workspace links.
  • Top-level viewer sections let users move between Overview, Graph, Memory, Intel, Review, and Data without cramming everything into one page.
  • The viewer now uses a graph-first drawer workspace so controls, Inspector, Repo Intelligence, review/proof, and tables open only when needed instead of shrinking the graph by default.
  • Scope, 2D/3D mode, search, Path Finder, and Inspector access are now available from a compact floating toolbar on the graph canvas.
  • The viewer now includes Path Finder for highlighting the shortest dependency path between two code nodes in 2D Canvas or 3D Space.
  • Path highlights are preserved through graph capping so selected code routes stay visible in dense repos.
  • The viewer Repo Intelligence cockpit now includes a Workspace Map section for package dependencies, route links, topic links, and cross-repo co-change pairs.
  • Workspace intelligence is now browsable as rows instead of only appearing as summary counts.
  • kage workspace now reports same-author, near-time cross-repo co-change links from local git history.
  • The CLI and viewer workspace summary show co-change counts alongside package, route, and topic links.
  • kage workspace now reports source-evidence based topic/event producer-consumer links across sibling repos.
  • Workspace contract intelligence remains local and deterministic: package dependencies, route string evidence, and topic string evidence only.
  • Code graph call edges now carry confidence and resolution metadata for TypeScript AST, generic static, and external index facts.
  • The viewer and kage code-graph context now expose call confidence instead of treating every call link as certainty.
  • kage cleanup-candidates now reports unused exports and internal-looking unused symbols in addition to unreferenced files.
  • Symbol candidates include confidence, reasons, and source location while remaining review input only.
  • Code graph route extraction now recognizes Rails, Laravel, Spring, Go routers, Rust routers, and ASP.NET.
  • Code graph route extraction now recognizes FastAPI/APIRouter, Flask, and Django route declarations.
  • Route params are normalized into Kage's cross-framework :param graph format.
  • New kage hook install/status/uninstall command for marker-delimited git post-commit refresh automation.
  • Viewer Repo Intelligence maps for ownership, modules, onboarding targets, architecture communities, execution flows, and blast radius.
  • Updated README, npm docs, website docs, and hosted viewer assets for the new workflow.
  • Repo-local memory packets, code graph, recall, viewer, and PR checks remain git-shareable.
