User-impact highlights

What changed for users

Pre-edit risk and impact checks kage risk and the viewer surface dependents, co-change partners, churn, ownership silos, and missing tests before you edit shared files.
Workspace recall across sibling repos kage workspace indexes a local parent directory and exposes package, route, topic, and git co-change links without copying memory.
Conservative cleanup review input kage cleanup-candidates reports unreferenced files, unused exports, and internal-looking unused symbols with confidence and reasons. It never deletes code.
Mixed-language route and call indexing The code graph now recognizes FastAPI, Flask, Django, Rails, Laravel, Spring, Go, Rust, and ASP.NET routes, plus confidence-scored call edges.
Optional post-commit refresh hook kage hook install adds a marker-delimited git hook that refreshes memory and branch summaries after commits, preserving existing hook content.
Viewer redesigned around tasks Overview, Graph, Memory, Risks, and Review pages with a focused inspector, search, and contextual path tracing — not a graph-engine cockpit.
