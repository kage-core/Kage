Why Kage

Three guarantees no other agent memory makes.

Every agent remembers now — the danger is acting on memory that's wrong. An agent that trusts a stale or hallucinated memory is worse than one with none. Kage is built so it can't.

✓ Validated on write

A memory that cites a file which doesn't exist is rejected — hallucinated knowledge never enters the store.

✓ Withheld on recall

When the code a memory cites is deleted or refactored, Kage withholds it from the agent and flags it for you under "Needs review."

✓ Grounded to your code

Memory links to your code graph, so recall returns the bounded blast radius of a change — not just matching text.

The session loop

It takes care of itself.

Memory systems die when they depend on anyone remembering to use them. Kage's loop runs on hooks — capture, resume, repair — with you reviewing, not babysitting.

Auto-capture, signal-gated

Sessions that produced no explicit learnings are distilled into pending drafts at session end. A signal score filters out raw payloads and noise — junk never becomes memory, and drafts never enter trusted recall until you approve them.

"Previously…" on every start

kage resume opens each session with what you were doing, what was learned, drafts awaiting review, and a compact timeline of recent memory — ~800 tokens, injected automatically.

Memory at the moment of reading

When the agent opens a file, verified packets citing that file are injected right then — checked against the code first, deduped per session, never blocking.

One-command recovery

Broken packet? Stale index? kage repair backs up, fixes, and rebuilds with a receipt of every step. Every CLI error ends with one copy-pasteable Try: command.

Memory that follows you

Every machine. Your own remote. No account.

Repo memory travels with the repo. Personal memory — your habits, your cross-project lessons — lives in ~/.kage/memory and syncs over a private git remote you already own.

personal memory + syncno account, no API key
kage learn --personal --learning "Always run the full suite before releasing"
kage sync setup --remote git@github.com:you/kage-memory.git   # once, any machine
kage sync     # pushed 2, pulled 1, resolved 0 — conflicts keep both versions

Recall labels personal packets [personal] and re-verifies their citations in every clone — synced memory obeys the same trust rules as everything else.

Quickstart

Start with one repo.

Requires Node.js 18 or newer. The package installs two binaries: kage and kage-graph-mcp.

quickstart npm 
# see the trust wedge in 30 seconds — no setup, no API key
npx -y @kage-core/kage-graph-mcp demo

# then wire it into your repo + agent — one command, auto-detects your agents
npx -y @kage-core/kage-graph-mcp install
kage setup verify-agent --agent claude-code --project .   # confirm it's live
