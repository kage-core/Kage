The problem

Agents forget the context your team keeps relearning.

Every repo has commands, gotchas, conventions, and decisions that are obvious only after hours of work. Kage gives that knowledge a durable place to live.

Before Kage

Every new session asks the same setup questions, scans the same files, and risks repeating the same mistakes.

With Kage

Agents recall source-backed memory before they act: commands, decisions, known bugs, and path-specific workflows.

For teams

Memory travels with the repo, so teammates and different agents start from the same reviewed context.

How it works

Capture what matters. Recall it when it matters.

Kage is intentionally boring infrastructure: a CLI, an MCP server, repo-local packets, a code graph, and retrieval tuned for agent work.

1
Initialize the repo`kage init` creates the local memory harness and source graph.
2
Agents learn during real workBug causes, setup commands, decisions, and gotchas become concise memory packets.
3
Memory stays reviewablePackets are git-visible JSON, not hidden chat transcripts or vendor state.
4
Future sessions recall contextAgents retrieve relevant packets and code graph facts before editing.
