# DESIGN: Kage's collaborative-memory story is git-native (v0), not a fake org tier — audit found promotion_requires_review and scope=org are unenforced

> Full audit 4 parallel passes over kernel.ts/proxy.ts/cli.ts/hooks/CI found the 'collaborative team memory' and org/gl…

Full audit (4 parallel passes over kernel.ts/proxy.ts/cli.ts/hooks/CI) found the 'collaborative team memory' and org/global promotion story was partly aspirational: quality.promotion_requires_review=true is written on every packet but never read/enforced anywhere in code (convention only, per kernel.ts:2378 comment); scope='org' exists in the MemoryScope type union but is never assigned; there is no kage_submit/global-graph implementation despite a shipped skill description claiming one; docs/CLOUD.md confirms the hosted team tier is v2, gated on a waitlist signal, not built. What IS real and already shared for free: .agent_memory/packets/*.md is git-tracked, so every teammate who clones the repo shares it via git; a real merge driver (git config merge.kage-packet.driver) runs on conflicts; a real interactive kage review queue exists for pending packets; the kage-sync CI bot keeps shared graphs fresh on every push to master. DECISION: don't build a fake org tier. Make the git-native team tier that already exists (repo-scoped, git-shared) actually trustworthy: no silent data loss on concurrent edits, real author attribution, and a legible team-health receipt (kage team). Selling point: 'Your team's memory already lives in your repo. Kage makes sure it's never wrong and never silently lost verified against your code on every read, attributed to whoever wrote it, legible in one command.' Full design + honest limitations (still last-write-wins not 3-way merge; proxy/hooks can still double-capture; no real-time/hosted tier) in docs/COLLABORATIVE_MEMORY.md.
Evidence: docs/COLLABORATIVE_MEMORY.md written with full audit citations. Shipped: mergePacketFiles() loser preservation, author_name attribution, kage team CLI command, proxy session_id fix. 391/391 tests pass including a simulated two-contributor end-to-end scenario (teamMemoryReport test).
Verified by: npm test: 391/391 pass; node dist/cli.js team --project . runs cleanly against this repo
