# kage skills: codify verified memory into git-native team SKILL.md files

> kage skills generateSkills in mcp/kernel.ts, CLI in mcp/cli.ts, MCP tool kage skills in mcp/index.ts distills durable…

`kage skills` (generateSkills in mcp/kernel.ts, CLI in mcp/cli.ts, MCP tool kage_skills in mcp/index.ts) distills durable verified memory into .claude/skills/<slug>/SKILL.md files agents auto-load. It is Kage's git-native answer to cloud team-memory tools (Hivemind/Deeplake): skills are plain files committed and reviewed in the same PR, no account/hosted store. Eligibility (isSkillWorthy): type must be runbook or workflow (SKILL_ELIGIBLE_TYPES — decisions excluded as rationale-only); excludes isGeneratedChangeMemory (per-diff packets) and looksLikePayloadMemory (leaked hook/tool payloads via SKILL_PAYLOAD_MARKERS); requires context.action OR body>=120 chars; and recallHardStaleReason must be null (grounded, non-stale) so a skill never teaches deleted code. Output dir defaults to .claude/skills (override --dir / dir). Each SKILL.md has name+description frontmatter and a 'Generated by kage skills' marker. GOTCHA: skill files must live in <dir>/<slug>/SKILL.md (own folder), never at repo root (npx skills add copies the whole repo if SKILL.md is at root). Since the value is team-sharing via git, generateSkills sets git_ignored (via git check-ignore) and the CLI warns when the output dir is git-ignored — note .claude/ is git-ignored in THIS repo, so Kage's own generated skills are not committed by default. Flags: --dry-run previews without writing.
Verified by: npm test --prefix mcp (260 passing incl 2 new skills tests); kage skills --dry-run on this repo yields 9 grounded procedures
