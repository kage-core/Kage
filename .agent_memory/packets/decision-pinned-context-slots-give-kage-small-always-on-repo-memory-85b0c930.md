# Pinned context slots give Kage small always-on repo memory

> After inspecting pinned context patterns, Kage now implements repo local pinned context slots instead of global perso…

After inspecting pinned context patterns, Kage now implements repo-local pinned context slots instead of global persona slots. Slots live in .agent_memory/slots/slots.json, are reviewable in git, block sensitive content, support CLI/MCP/REST create/list/delete, and are injected into recall/context before task-specific memory. Refresh writes .agent_memory/reports/context-slots.json and the viewer dashboard shows a Pinned context card. This gives agents tiny always-relevant repo guidance without loading all memory or relying on hidden global state.
