# Contradiction detector: require distinctive-subject paraphrase, not generic-token overlap

> detectContradictions mcp/kernel.ts had two precision holes that produced 786 phantom conflicts on this repo: 1 the su…

detectContradictions (mcp/kernel.ts) had two precision holes that produced 786 phantom conflicts on this repo: (1) the subject gate 'if subjectSimilarity < threshold && subjectOverlap < 2 continue' let any two packets sharing a cited path plus 2 GENERIC title/summary tokens through, and tokenSet does not strip stopwords or domain words; (2) opposingPolarity fires on negation-cue ASYMMETRY (one body contains 'removed'/'deprecated'/'never', the other does not), which is incidental not contradictory. Fix: added CONTRADICTION_GENERIC_SUBJECT (English stopwords + house-style words kage/mcp/memory/agent/code/graph/decision/etc) and distinctiveSubjectTokens(); compute subjectSimilarity over DISTINCTIVE tokens; raise subjectThreshold 0.34->0.5; REMOVE the subjectOverlap bypass entirely (a real contradiction is the same claim negated, i.e. a paraphrase). Also cleared stale quality.contradicts flags the old detector stamped on packets. Do NOT reintroduce a token-count bypass — that was the false-positive door.
Verified by: kage conflicts 786->0 on this repo; 342 tests pass incl new regression test
