# Stale-catch: change-time invalidation heartbeat (staleCatch, kage staleguard)

> Stale catch is the change time retention heartbeat: staleCatch projectDir, changedFiles? in mcp/kernel.ts intersects …

Stale-catch is the change-time retention heartbeat: staleCatch(projectDir, changedFiles?) in mcp/kernel.ts intersects working-tree changes (git diff HEAD + staged + untracked, or an explicit file list) with packet path_fingerprints and reports each mismatch as {packet_title, packet_id, cited_path, reason}. Every catch records a value-ledger event kind stale_caught (totals + window summaries, shown by kage gains as 'stale caught at change-time'). Surfaces: kage pr check prints the human summary first, kage staleguard is the advisory exit-0 pre-commit/pre-push variant, and the kage_pr_check MCP tool prepends the same lines via formatStaleCatch so agents relay it. The ValueEvent union, ledger totals, and ValueWindowSummary all gained stale_caught; extend all of them together when adding a new ledger kind.
