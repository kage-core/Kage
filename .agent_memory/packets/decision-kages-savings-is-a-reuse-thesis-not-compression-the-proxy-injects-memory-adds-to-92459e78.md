Kage's "savings" is a REUSE thesis, not compression: the proxy INJECTS memory (adds tokens to one prompt); the claim is the agent avoids re-reading/re-deriving across a task. Reporting must keep MEASURED and ESTIMATED distinct. value_receipt.tokens_saved and the kage gains token/$ figures are ESTIMATES (per-recall read-vs-source / discovery-cost heuristic in recallTokensSaved/replayTokensSaved); stale_withheld, recalls, stale_caught and caller_answers are MEASURED counts. Rendering now lives in kernel.ts formatValueGains / formatRecallValueReceipt (unit-tested in kernel.test.ts) so the honesty contract can't drift back to the old "saved you N tokens" phrasing. The deterministic reuse eval is benchmarks/reuse-value-kage.mjs (answer_from_memory_rate is a PROXY: does the recalled memory BODY carry the labeled fact so the agent skips the cited files; baseline 1.0 / files_avoided 8 / false_help_rate 0). Its --receipts mode reads real vNext receipts via readLocalReceipts/tokenDelta/costDelta: on this repo's 8 audit/assist receipts the injected BYTES are measured (+38,737) but the injected TOKEN delta is unavailable because no receipt is measured on BOTH sides (audit measures only before, assist only after). The end-to-end token delta (agent with-memory vs without) still needs a live A/B run and is deliberately not claimed. Bytes are never converted to tokens.
