# Tree-sitter tier sits between TS-AST and regex extraction

> Per file structural extraction in kernel.ts now has three tiers: TypeScript compiler AST for TS/JS, tree sitter web t…

Per-file structural extraction in kernel.ts now has three tiers: TypeScript compiler AST for TS/JS, tree-sitter (web-tree-sitter WASM) for python/go/rust/java/ruby, and the regex extractGenericSymbols/extractGenericCalls for everything else (kotlin, php, csharp, cpp, swift) or when a grammar fails to load. Because web-tree-sitter only initializes asynchronously while buildStructuralIndex/buildCodeGraph stay synchronous, grammars are loaded once per process before any file loop: cli.ts main() awaits ensureTreeSitterLanguages() after the help branch, index.ts callTool() awaits it first, and structural-worker.ts awaits it for the languages in its batch before mapping files. extractTreeSitterSymbols/extractTreeSitterCalls return null when no parser is loaded and callers fall back to the regex tier (?? extractGenericSymbols / ?? extractGenericCalls). Tree-sitter call edges use resolution "tree_sitter_name" with resolveCallTargets weights {local 0.8, imported 0.75, sameDir 0.55, nameOnly 0.32} — between regex (0.7) and TS-AST (0.9). usableStructuralCache treats cached generic-static symbols as a miss once the language's grammar is loaded so regex-tier cache entries cannot pin weak symbols past the upgrade. STRUCTURAL_EXTRACTOR_VERSION=4 and CODE_GRAPH_BUILDER_VERSION=3 force cache rebuilds.
Evidence: mcp/kernel.ts extractTreeSitterSymbols/extractTreeSitterCalls/usableStructuralCache; kernel.test.ts tree-sitter python and go tests.
