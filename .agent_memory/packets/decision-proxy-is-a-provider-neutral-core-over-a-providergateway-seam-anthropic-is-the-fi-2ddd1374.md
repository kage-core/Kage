# Proxy is a provider-neutral core over a ProviderGateway seam; Anthropic is the first gateway

> The kage proxy mcp/proxy.ts is now a PROVIDER NEUTRAL core that dispatches to ProviderGateway implementations mcp/vne…

The kage proxy (mcp/proxy.ts) is now a PROVIDER-NEUTRAL core that dispatches to ProviderGateway implementations (mcp/vnext/adapters/gateway.ts). To add a provider (OpenAI Task 2, Gemini Task 3), append a new gateway to `defaultGateways` — do NOT touch the core. The seam interface: matches(method,path,body), parseRequest(body)->{model,systemText,lastUserText}, inject(body,memoryText)->{body,applied}, extractUsage(responseBody), createTokenCounter(opts)|null (null when no cheap count-tokens endpoint => receipt stays honestly 'partial'), buildReceipt(input), captureEvents(ctx)->EvidenceEvent[], priceSnapshots. The core owns: eligibility dispatch (selectGateway), fail-open, byte-identical audit forward, streaming passthrough, receipt+delivery recording, evidence emission, workspace routing. All Anthropic wire semantics (path eligibility isCompletionsRequest, last-user-turn injection, response tool_use parsing, count_tokens probe, ANTHROPIC_PRICE_SNAPSHOTS) live in anthropicGateway and mcp/vnext/adapters/anthropic-proxy.ts. Capture unification: the proxy emits protocol-v1 evidence to the runtime /v2/events via the fail-open adapter client (readAdapterConnection + sendAdapterEvent) AFTER clientRes.end() inside a catch-all, exactly like recordReceipt — a dead runtime never changes the response. The protocol-v1 event assembly + source_fingerprint (assembleEvidenceEvent reuses claudeSourceFingerprint/claudeTaskIdentity) is SHARED with the hook adapter so duplicate posts dedupe on source_fingerprint; only the payload is provider-specific. injectMemory/isCompletionsRequest stay exported from proxy.ts (thin bindings/re-export) for test + caller compatibility.
Evidence: Refactor kept every existing proxy test byte-for-byte green; new tests: seam-dispatch to Anthropic gateway, response tool_use -> tool_result evidence, prompt evidence lands via real startLocalRuntime+startProxy, fail-open when evidence runtime down.
Verified by: npm test --prefix mcp (669/669 + dogfood 12/12); npm run test:vnext (224/224); node --test proxy.test.js + adapter.test.js (73/73)

## Verification
