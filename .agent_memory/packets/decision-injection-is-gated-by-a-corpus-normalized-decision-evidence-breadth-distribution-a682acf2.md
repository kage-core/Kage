# Injection is gated by a corpus-normalized decision: evidence breadth + distribution spike; pong problem measured to zero

> W3 commit 8b80f3c resolved the negative result "recall scores are not corpus normalized; an absolute injection floor …

W3 (commit 8b80f3c) resolved the negative result "recall scores are not corpus-normalized; an absolute injection floor is impossible". The impossibility claim was CORRECT for absolute floors — the fix is a RELATIVE decision computed where the full distribution exists. recall() now returns RecallResult.injection (decideRecallInjection in mcp/kernel.ts) requiring BOTH: (1) EVIDENCE BREADTH — the top candidate must match >= 2 distinct meaningful (len>=3) query terms (countDistinctTermMatches); a content-free prompt like "Reply with the single word: pong" spikes a websocket-heartbeat runbook on ONE token with a score as high as a real question's, but cannot match several distinct terms because it does not contain several; (2) CORPUS NORMALIZATION — the top must SPIKE above this corpus's own score band (z over the full candidate list blended with runner-up lead; on tiny corpora with <5 candidates, runner-up gap rules with a modest single-candidate anchor of 6 since breadth already removed one-term accidents). composeInjection (mcp/proxy.ts) gates on the decision, then dominance-trims co-attached packets to score >= 0.5×top. Measured on the acceptance harness (benchmarks/injection-relevance-kage.mjs, drift-checked against the real production path; baseline recorded in-file, --assert-baseline green): false_injection 0.6667→0 (large store 1.0→0), absent_injection 0.875→0, precision 0.1538→0.6364, small_store_recall and expected_top_hit_rate held at 1.0. GOTCHA: the harness's drift check compares GATE agreement, not exact attach counts — production recall runs after earlier cases recorded access boosts (order-dependent by design), so the 0.5×top trim count can wobble by a packet under an agreeing gate.
Evidence: bench:injection measured: false 0.6667→0, absent 0.875→0, precision 0.1538→0.6364, recall/top-hit 1.0 held, errors 0, --assert-baseline exit 0. Suite 1460/1460. Commit 8b80f3c.
