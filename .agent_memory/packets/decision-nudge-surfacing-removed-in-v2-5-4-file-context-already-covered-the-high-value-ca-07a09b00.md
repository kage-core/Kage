# Nudge surfacing removed in v2.5.4 — file-context already covered the high-value case

> The watcher "nudge" surfacing — added in 2.5.2 prompt context and 2.5.3 edit time file context — was REMOVED in v2.5.…

The watcher "nudge" surfacing — added in 2.5.2 (prompt-context) and 2.5.3 (edit-time file-context) — was REMOVED in v2.5.4. surfacePendingNudges and its wiring into the prompt-context and file-context CLI commands (mcp/cli.ts, mcp/kernel.ts) are deleted. Rationale: the high-value behavior ("here's verified memory for the file you're about to touch") is already delivered by the file-context PreToolUse(Read/Edit) hook injecting the packets that cite the file, so the nudge channel was redundant for that case; and it had no generator for the rest — nothing auto-runs the kage-watcher, so .agent_memory/nudges/pending.jsonl stayed empty in practice. What was NOT removed and is still current: recall (prompt-context), file-context citation injection, capture, the shell-paste capture guard (isSerializedDumpTitle rejects "user@host dir % cmd" titles), and the normalizeRepoPath / trailing-slash cleanup to file-context path matching. The kage-watcher agent definition is now ingest-only. Supersedes the two nudge-surfacing packets.
Verified by: 353 unit + 12 dogfood tests pass; end-to-end through global 2.5.4 binary (file-context and prompt-context ignore pending.jsonl, inbox left untouched); published v2.5.4
