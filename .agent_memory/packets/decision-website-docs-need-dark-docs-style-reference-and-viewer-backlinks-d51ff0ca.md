# Website docs need dark docs-style reference and viewer backlinks

> The Kage website should keep a dark terminal/memory graph identity rather than a generic white OSS landing page. Docs…

The Kage website should keep a dark terminal/memory-graph identity rather than a generic white OSS landing page. Docs must be a real documentation shell with sidebar navigation and full CLI/MCP reference, not just setup and quickstart. The hosted viewer must include visible links back to Home, Docs, Releases, and GitHub so it is not a dead-end tool page. The source for those viewer links lives in mcp/viewer; docs/viewer is generated by the Pages workflow and should not be committed.
Evidence: Updated homepage/releases to dark theme, rebuilt docs/guide.html as sidebar reference covering CLI and MCP methods, and added viewer site links in mcp/viewer.
Verified by: Chrome local preview; git diff --check; curl local pages; npm test --prefix mcp
