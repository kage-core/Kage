# Reference recall should stay text-led for external memory benchmarks

> Reference memory packets should not let graph, path, and tag boosts swamp direct lexical evidence. LongMemEval S show…

Reference memory packets should not let graph, path, and tag boosts swamp direct lexical evidence. LongMemEval-S showed reference packets rank best when BM25/text evidence leads, graph/path boosts are disabled for reference packets, and a body-only BM25 fallback is used for raw transcript-style references. Current full LongMemEval-S retrieval after the tuning: R@5 96.17%, R@10 98.30%, R@20 99.79%, MRR 0.9110, NDCG@10 0.9279 over 470 non-abstention questions.
Evidence: Implemented reference-specific graph/path boost caps plus body-only BM25 fallback in mcp/kernel.ts, added regression coverage in mcp/kernel.test.ts, and documented benchmark results in benchmarks/LONGMEMEVAL.md.
Verified by: npm test --prefix mcp; node benchmarks/longmemeval-kage-retrieval.mjs --data /private/tmp/kage-external-bench/longmemeval_s_cleaned.json --limit 470 --top-k 10
