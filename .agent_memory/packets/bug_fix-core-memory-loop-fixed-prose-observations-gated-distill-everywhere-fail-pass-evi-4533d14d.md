# Core memory loop fixed: prose observations, gated distill everywhere, fail→pass evidence crosses promote bar

> The end to end memory loop session 2 inheriting session 1's learning was broken by a causal chain, fixed 2026 07 03 i…

The end-to-end memory loop (session 2 inheriting session 1's learning) was broken by a causal chain, fixed 2026-07-03 in mcp/kernel.ts: (1) The PostToolUse hook template serialized tool payloads via json.dumps, which observationSignalScore's jsonNoiseText hard-rejects — so ALL tool observations scored 0 and Edit's new_string (where fixes/conventions live) was discarded whenever tool_response was truthy. Fixed: the hook's python builder now extracts prose — Edit/Write use tool_input old→new excerpts, Bash uses command + stdout/stderr tail, never json.dumps of dicts. (2) observe.sh ran `kage distill` WITHOUT --auto on SessionEnd/PreCompact/SubagentStop — the ungated manual path with no signal filter, no dedupe, pendingReview=false, which is how 116KB junk got stamped approved+verified. Fixed: --auto added, and manual mode is now safe anyway (signal gate + sessionAlreadyCaptured dedupe run in BOTH modes, all distilled drafts born pending). (3) Grounded drafts scored 70 < the 72 auto-promote bar and died in the pending inbox recall never reads. Fixed: distillSession detects a fail→pass pair on the same command in the session's observations and appends a truthful "Verified: `cmd` failed then passed — reproduced" line, which carries the admission verification bonus; plus debugging keywords (fail/fix/error/broken/regression) added to reusablePromptObservation so debugging prompts survive the gate. Also: observation_ids capped at 20 (was 112KB arrays), prompt text clamped 500/4000, isSerializedDumpTitle catches "Runbook: Tool failed: {...}". Acceptance test (scratchpad loop-acceptance.sh pattern): two-session sandbox through real hooks with kage absent from PATH — session 2's prompt-context now contains session 1's convention verbatim; before the fix it received zero. Remember: hook template changes require `kage gen-plugin-hooks` (drift test enforces it).
Evidence: Live two-session sandbox: PASS (convention injected into session 2's additionalContext); pre-fix tracer run showed zero inheritance and skipped_low_signal:3 on identical material
Verified by: npm test 376/376 + 12/12; loop-acceptance.sh PASS 2026-07-03
