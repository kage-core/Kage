# Viewer defaults favor focused pages over diagnostic walls

> The Kage viewer should keep normal pages sparse and task focused: Overview is capped to three top metrics/charts/card…

The Kage viewer should keep normal pages sparse and task-focused: Overview is capped to three top metrics/charts/cards, Graph shows the canvas plus search/action controls only, Memory hides overview charts and governance by default, Risks prioritizes actual change-risk cards, and Review hides proof/benchmark panels. Diagnostics such as raw artifacts, path tracing, status strips, and proof surfaces should stay out of the default workflow unless explicitly opened.
