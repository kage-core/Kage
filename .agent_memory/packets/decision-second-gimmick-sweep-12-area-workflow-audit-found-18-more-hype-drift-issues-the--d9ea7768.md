# Second gimmick sweep: 12-area Workflow audit found 18 more hype/drift issues the first pass missed

> The first gimmick removal pass commit 57a18a3 only covered docs/index.html, docs/releases.html, README.md, CONTRIBUTI…

The first gimmick-removal pass (commit 57a18a3) only covered docs/index.html, docs/releases.html, README.md, CONTRIBUTING.md, and part of mcp/cli.ts/kernel.ts. A user follow-up ('i asked to remove gimmicks for the project') triggered a full 12-area Workflow sweep (12 parallel finders + one independent adversarial judge per candidate, each judge required to re-read the file itself rather than trust the finder) across every area the first pass never touched: docs/guide.html, docs/demo.html, docs/benchmarks.html, docs/BENCHMARKS.md, docs/CHECK_VALIDATION.md, docs/CLOUD.md, docs/COLLABORATIVE_MEMORY.md, docs/MEMORY_ADMISSION.md, docs/TRUST.md, docs/scorecard-metrics.md, AGENTS.md, ROADMAP.md, CODE_OF_CONDUCT.md, OKF_STANDARD.md, CHANGELOG.md, mcp/viewer/{index.html,console.js}, mcp/package.json, skills/kage-memory/SKILL.md, plugin/{commands/*.md,.claude-plugin/plugin.json,.codex-plugin/plugin.json}, mcp/cloud-server.ts's dashboard HTML, the rest of mcp/cli.ts, and both benchmark/ READMEs.

Result: 35 candidates judged, 18 definite gimmicks, 3 ambiguous (resolved by hand), 14 correctly kept as legitimate (real computed benchmark numbers, a tested ~60s ceiling claim validated against a 50k-commit external repo, honest self-graded-benchmark disclosures, etc.) -- the judges did real verification work (ran the actual CLI, git-blamed claims, cross-referenced other docs) rather than rubber-stamping.

Fixes applied: docs/guide.html (stale 'v2.2' version stamp, an unbacked 60-second timing claim, ambiguous 'Cloud waitlist' footer link -> 'Hosted Cloud waitlist'), docs/benchmarks.html ('no other tool measures' -> scoped mechanism claim), docs/BENCHMARKS.md ('and that we lead on' contradicted the doc's own no-head-to-head policy), docs/scorecard-metrics.md (dropped 'single best argument for installing Kage' superlative and the same 60-second claim), docs/COLLABORATIVE_MEMORY.md (dropped a stale '391/391' test count -- actual is 402/402+12/12), docs/TRUST.md (the 'Reference result' block was a frozen one-time snapshot with a now-wrong packet-count fraction; added a commit stamp and reproduction instructions instead of re-freezing a new number), OKF_STANDARD.md ('200/200' was stale -- actual is 203 -- and falsely cited okf.test.ts as verifying an ongoing count when it only covers single-packet round-trips; reworded to historical framing), ROADMAP.md (a 'Now' bullet was contradicted by the very commit that introduced it -- guide.html was already caught up; reworded to name the real gap, README.md's silence), CHANGELOG.md (the v2.5.2-2.5.4 entries used 'kage-watcher' as if the reader already knew it -- it was actually a maintainer-only local subagent never shipped in the npm package; added a defining parenthetical and softened 'felt behavior' framing that oversold a feature whose only data source never ran for real users), mcp/viewer/console.js + docs/viewer/console.js (byte-identical mirrors -- 'nothing in the graph is a guess' overclaimed against the system's own confidence field; reworded to 'trace back to an observed session'), plugin/commands/scan.md (frontmatter and body gave two different, both-wrong severity orderings for the 8 finding categories, contradicting the real cli.ts order and omitting 3 of 8 categories -- fixed to match), plugin/.claude-plugin/plugin.json + plugin/.codex-plugin/plugin.json + mcp/index.ts (three different version numbers -- 2.5.5, 2.0.1, 2.0.0 -- for the same lockstep-versioned product; brought all three, plus mcp/package.json, to 3.3.0), mcp/cli.ts (the install command's 'Kage then recalls automatically every session' printed unconditionally even when zero agents were wired or a non-claude-code agent was wired without ambient hooks -- now conditioned on what actually happened; dropped an unbacked '60-second'/'unusually well distributed' framing in 4 places; added a gloss for the unexplained 'caller answers' metric in kage gains, matching the existing 'Knowledge replay value' precedent).

Ambiguous items resolved by hand rather than left to a judge verdict: the demo.html 'We reply same day' SLA-style promise (unverifiable maintainer behavior claim) was softened to 'We usually reply within a day' in both the form and success-message copy, since a hard guarantee about future personal response time can't be backed by anything in the repo.
Evidence: docs/guide.html, docs/demo.html, docs/benchmarks.html, docs/BENCHMARKS.md, docs/scorecard-metrics.md, docs/COLLABORATIVE_MEMORY.md, docs/TRUST.md, OKF_STANDARD.md, ROADMAP.md, CHANGELOG.md, mcp/viewer/console.js, docs/viewer/console.js, plugin/commands/scan.md, plugin/.claude-plugin/plugin.json, plugin/.codex-plugin/plugin.json, mcp/index.ts, mcp/cli.ts
Verified by: npm test: 402/402 + 12/12 pass; kage check: 0 confirmed drift; live CLI smoke test of install/scan/gains output
