# Phase D Task 6: honest agent-surface capability certification

> Phase D Task 6: honest agent surface capability certification. certifySurface mcp/vnext/adapters/capability matrix.ts…

Phase D Task 6: honest agent-surface capability certification. certifySurface (mcp/vnext/adapters/capability-matrix.ts) labels a surface "automatic" ONLY when a transcript-based smoke test proves the requested injection sentinel appears in the transcript — never from installed config, and captured telemetry (which proves capture) never implies injection. Each surface has a declared injection CEILING in SURFACE_PROFILES; a sentinel can confirm the ceiling but never raise a surface above it. Codex's ceiling is mcp_fallback because Codex plugins do not establish a Kage-owned session injection hook, so Codex is automatic capture + honest mcp_fallback and counts_as_automatic_attachment is always false, even if a sentinel is present. counts_as_automatic_attachment = capture==automatic AND injection in {automatic_task,automatic_session,gateway}. fixture_fingerprint is sha256 over the meaningful inputs only (not certified_at) so it is deterministic; certified_at accepts an override for reproducibility. kernel.ts agentSurfaceCertificationGate requires REQUIRED_AUTOMATIC_SURFACES = [claude-code, anthropic-proxy, cursor] to certify automatic; Codex is visible but not required and never fails the gate; a failing Cursor cert keeps the gate failed instead of relabeling. cursor-hooks.ts/codex-otel.ts generate plugin/cursor/hooks.json + plugin/codex/otel-config.toml deterministically. Committed e9316933.
