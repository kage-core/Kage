# kage proxy injection is now BODY-AWARE: top 2 hits inject full packet body, rest stay pointers

> Changed proxy injection from summary only to body aware. Before: injectMemory used recall's context block, which emit…

Changed proxy injection from summary-only to body-aware. Before: injectMemory used recall's context_block, which emits only title + one-line summary + provenance per packet — so for implementation questions the agent still re-read the code (the formula was never in the summary). After: new pure buildInjectedMemory(results) emits the FULL body of the top BODY_PACKET_COUNT (2) hits, capped MAX_BODY_CHARS (2200) each, and keeps the rest as one-line pointers; whole thing still capped at MAX_MEMORY_CHARS (6000). Verified against the real repo: querying 'how does kage savings compute the percent' now injects the savings packet's actual mechanism text (baseline=full-file reads, with-Kage=recall+code-graph, aggregate to context_reduction_percent, 97%) instead of just its positioning summary. This is what makes the proxy feel like the agent KNOWS the repo rather than just being pointed at it. Honest limit: bodies describe decisions/rationale, so deep line-level questions still need the code.
Evidence: node -e recall+buildInjectedMemory on this repo: 4 hits, 4073 injected chars, 2 full-body sections; first 1400 chars contain the savings mechanism.
