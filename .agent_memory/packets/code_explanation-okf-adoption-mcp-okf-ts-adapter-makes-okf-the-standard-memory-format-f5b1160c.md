# OKF adoption: mcp/okf.ts adapter makes OKF the standard memory format

> Kage adopted Google's Open Knowledge Format OKF as its standard on disk/interchange memory format. Implementation liv…

Kage adopted Google's Open Knowledge Format (OKF) as its standard on-disk/interchange memory format. Implementation lives in mcp/okf.ts (a pure, self-contained adapter; zero new deps). A MemoryPacket renders as an OKF concept .md = YAML frontmatter (OKF `type` required + recommended title/description/resource/tags/timestamp) where Kage's trust metadata rides in OKF-legal custom `x-kage-*` frontmatter keys (x-kage-id/type/status/scope/visibility/confidence/verified/paths/stack), plus a readable body (heading, prose, ## context sections, # Citations). Lossless round-trip is achieved with a fenced ```json kage-state block in the body that embeds the exact packet — packetToOkfConcept→okfConceptToPacket is an identity when that block is present. Concepts WITHOUT a kage-state block (foreign/hand-authored OKF) are imported best-effort from frontmatter+body, which lets Kage consume any third-party OKF bundle. Key fns: packetToOkfConcept, okfConceptToPacket, migratePacketsToOkf (writes .agent_memory/okf/<type-slug>/<slug>-<idhash8>.md + reserved index.md/log.md), loadOkfConcepts, lintOkfConcept/lintOkfBundle, okfType/kageType (TYPE_DISPLAY map: e.g. bug_fix↔"Bug Fix"). CLI: `kage okf migrate|lint|import` wired in mcp/cli.ts (command==="okf"; subcommand is args[1] since args[0] is the command). yamlScalar uses JSON.stringify to dodge all YAML quoting edge cases. Verified: 360/360 unit tests pass (mcp/okf.test.ts adds 5), and `kage okf migrate` on this repo converted 200 approved packets into a conformant OKF bundle that lints clean and round-trips byte-exact (200/200, mismatch=0). Standard documented in OKF_STANDARD.md + README Storage section. NOT yet done: making the generated AGENTS.md/CLAUDE.md policy reference OKF, and switching the primary read/write path (writePacket/loadPacketsFromDir) to .md (currently OKF is a verified export/import layer alongside the JSON packet store).
Evidence: npm run build clean; node --test dist/*.test.js → 360 pass 0 fail; node dist/cli.js okf migrate → 200 concepts; okf lint → all conformant; round-trip check loadApprovedPackets vs loadOkfConcepts → exact=200 mismatch=0 missing=0.
