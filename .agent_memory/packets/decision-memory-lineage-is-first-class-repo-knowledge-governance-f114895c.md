Use kage supersede --packet <old-id> --replacement <new-id> --reason <why> when a newer packet replaces old repo knowledge; the old packet becomes superseded, recall ignores it, bidirectional lineage edges are written, and kage lineage plus the viewer Memory page show current replacements and orphaned retired packets.
