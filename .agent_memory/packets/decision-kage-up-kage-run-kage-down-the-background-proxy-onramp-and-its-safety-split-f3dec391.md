# kage up / kage run / kage down: the background-proxy onramp and its safety split

> kage up, kage run, and kage down mcp/cli.ts dispatch; logic in mcp/vnext/runtime/commands.ts + proxy daemon.ts are th…

kage up, kage run, and kage down (mcp/cli.ts dispatch; logic in mcp/vnext/runtime/commands.ts + proxy-daemon.ts) are the one-command onramp for the proxy-primary path. Invariants future agents must preserve: (1) upProject() is still a spawn-free PLAN — connectProject (byte-idempotent audit-only config; --mode can never leak into the config on disk) plus the VERIFIED daemon-state read plus the port probe; the CLI performs the actual start. The proxy now starts DETACHED in the background by default (startProxyDaemon: spawn detached with stdio to proxy.log, poll probeLocalPort, write .agent_memory/daemon/proxy.json only after liveness is confirmed); --foreground keeps the old terminal-owning behavior and writes no daemon state, so kage down does not manage it. (2) up defaults to --mode audit while bare kage proxy keeps assist for back-compat — deliberate asymmetry, do not "fix" it. (3) up is idempotent through the verified state: a VERIFIED running proxy (pid alive+ours+node AND port accepting — never the file alone) is reused with ITS port/mode and exit 0, with an honest warning if flags differ; a stale record is cleaned + reported + started fresh; an unrecorded listener on the port keeps the can't-tell message suggesting --port. (4) kage run sets ANTHROPIC_BASE_URL (and nothing else), propagates the child exit code, ignores SIGINT while the child runs, NEVER auto-starts the proxy, and with no --port auto-discovers the verified recorded port before falling back to 8788. (5) kage down stops the whole stack symmetric with up: verified SIGTERM -> bounded grace -> SIGKILL for the proxy, state file removed only once the pid is really gone, runtime stopped via the kage daemon stop path (runtimeDownFrom), per-component honest output, exit 0 when the end state is nothing-running. (6) startProxy/startup banners stay mode-honest: audit may not claim injection. Reboot gap is documented, not hidden: no system service; one kage up after a reboot.
Evidence: 29 new tests (14 proxy-daemon lifecycle unit tests; 15 in commands.test.ts incl. a real end-to-end kage up -> verified state -> run auto-discovery -> down against the actual kage proxy child, plus a --foreground no-state test). Full suite 784/784 + dogfood 12/12; kage check 0 confirmed drift. Manual smoke: fresh up (pid in output), reuse ("already running (pid, port, mode)"), SIGKILL-stale cleanup note + fresh start, down per-component output, down-again no-op. Commit 8983d2b.
Verified by: npm test --prefix mcp (784 pass, 0 fail) + manual lifecycle smoke on a scratch project (port 18788)
