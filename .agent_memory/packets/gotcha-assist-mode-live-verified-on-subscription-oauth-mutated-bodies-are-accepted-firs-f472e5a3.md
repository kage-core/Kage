# ASSIST mode live-verified on subscription OAuth: mutated bodies are accepted, first delivered deliveries recorded

> SETTLES the last open live question from the OAuth proxy saga: ASSIST mode — where the proxy MUTATES the request body…

SETTLES the last open live question from the OAuth proxy saga: ASSIST mode — where the proxy MUTATES the request body by appending verified repo memory to the last user turn — is accepted by Anthropic on a Claude Code SUBSCRIPTION (OAuth, not API key). Live-verified 2026-07-16 with a real headless Claude Code session (claude -p) through kage proxy --mode assist: 4 memories injected per eligible request, requests succeeded (no 401/429 from the mutation), the session answered normally, and the store recorded the first real "delivered" context deliveries (attachment_by_provider.anthropic delivered:2).

Combined with the same-day audit-mode verification (byte-identical forwarding, receipts 3→6, honest skips), the ORIGINAL 2026-07-07 hypothesis — that OAuth rejects mutated bodies with a 429 — is now DISPROVEN for last-user-turn injection. The historical 429s were genuine rate limiting. The one mutation that does remain forbidden is the SYSTEM prompt: injection deliberately targets the last user turn because OAuth identity checks are believed to cover `system` (never re-tested, and no reason to — the design never touches system).

Validation technique worth reusing: a headless `claude -p --model haiku "<one-shot prompt>"` with ANTHROPIC_BASE_URL pointed at the proxy is a cheap, complete end-to-end test — it exercises real OAuth auth, the full request/stream path, recall/injection, receipts, and deliveries in one command, with `kage status --project <dir> --json` as the assertion surface. Audit and assist can be validated against separate proxy ports on the same project store.
Evidence: Proxy stats: 2 req, 8 memories injected, 2 receipts, 54,078 in / 218 out tokens; the session completed normally (answered the prompt) with no 4xx from the mutated requests. kage status afterwards: attachment delivered:2 / skipped:6 / failed_open:0, attachment_by_provider.anthropic delivered:2 — the first real DELIVERED context deliveries. Same day, audit mode was validated the same way (receipts 3→6, all skipped) through a separate live session.
Verified by: Live run 2026-07-16: kage proxy --mode assist --port 8789 fronting api.anthropic.com; a real headless Claude Code session (claude -p, v2.1.211, subscription OAuth) driven through it end to end
