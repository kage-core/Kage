# Anchors are code identifiers; hooks carry version stamps; usage telemetry is live (ranks 6/8/10)

> Three deep dive fixes landed 2026 07 03. ANCHORS rank 6 : memoryPathFingerprints now extracts anchor candidates via c…

Three deep-dive fixes landed 2026-07-03. ANCHORS (rank 6): memoryPathFingerprints now extracts anchor candidates via codeAnchorTokens (camelCase, snake_case, backticked, or called() tokens, length>=6) instead of identifierTokens which matched every prose word — and symbol kinds are filtered (ANCHOR_SYMBOL_KINDS: function/class/method/interface/type/enum; constants only when the name contains an underscore), so packets stop anchoring to incidental symbols named "verified"/"same"/"when". Existing packets re-anchor whenever they are rewritten (reverify/capture); no mass migration was run. HOOK STAMPS (rank 10): every generated hook script carries "# kage-hooks-v<N>" (KAGE_HOOKS_VERSION, bump on template changes); claudeAmbientHookSummary reports an `outdated` list and ready:false when installed scripts are older, so doctor/verify-agent surface stale hooks instead of reporting ready:true — `kage hook install` / `kage setup claude-code --write` rewrites them. USAGE TELEMETRY (rank 8): refreshPacketStaleness reconciles the machine-local memory-access report (uses_30d/total_uses/last_accessed_at) onto packet.quality during refresh (quiet-branch gating applies), and kageFileContext serves now recordRecallAccess — resume deliberately does NOT count (it serves titles only). IMPORTANT correction to the audit: after reconciliation 131 packets show nonzero uses_30d (test runbook: 13) — memory WAS being served; the zero counters were a reconciliation gap, not proof of a dead loop. OKF import needed no change: the kage-state fence already round-trips full quality; zero-fill only applies to foreign concepts.
