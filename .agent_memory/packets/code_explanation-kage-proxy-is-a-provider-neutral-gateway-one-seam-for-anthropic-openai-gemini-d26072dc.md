# Kage proxy is a provider-neutral gateway: one seam for Anthropic, OpenAI, Gemini

> Kage's proxy is now a PROVIDER NEUTRAL gateway, not an Anthropic only proxy. The direction 2026 07 15 : the proxy is …

Kage's proxy is now a PROVIDER-NEUTRAL gateway, not an Anthropic-only proxy. The direction (2026-07-15): the proxy is Kage's primary path across every provider because it needs zero per-agent wiring; the Claude hook adapter is a secondary richer-signal path.

Architecture: mcp/proxy.ts is the provider-neutral core (listen/route, eligibility dispatch, fail-open, byte-identical audit forward, streaming passthrough, receipt+delivery recording, evidence emission to /v2/events, workspace routing). Each provider is a `ProviderGateway` (mcp/vnext/adapters/gateway.ts) implementing matches/parseRequest/inject/extractUsage/createTokenCounter/buildReceipt/captureEvents/priceSnapshots. Three are registered in `defaultGateways`: anthropicGateway (/v1/messages), openaiGateway (/v1/chat/completions + /v1/responses), geminiGateway (/models/{model}:generateContent + :streamGenerateContent). selectGateway is first-match-wins; the three matchers are provably disjoint. Adding a provider = one new gateway file, no core fork.

THE CENTRAL CORRECTNESS RULE — each provider reports token usage DIFFERENTLY, and getting it wrong recreates the "fake 98% saving" bug per provider. The receipt's before/after must be the SAME quantity (total prompt tokens incl cache):
- Anthropic: usage.input_tokens is the UNCACHED REMAINDER; total = input_tokens + cache_creation_input_tokens + cache_read_input_tokens.
- OpenAI: usage.prompt_tokens is the FULL prompt incl cached; prompt_tokens_details.cached_tokens is a subset. total = prompt_tokens directly.
- Gemini: usageMetadata.promptTokenCount is the FULL count incl cached; cachedContentTokenCount is a subset (ABSENT not 0 on cache miss). total = promptTokenCount directly.
Each was source-verified against the live API docs, not assumed. The neutral ProviderUsage decomposes each into {uncached, cache_read, cache_creation, output} so totalPromptTokens reconstructs the whole prompt on any measured side.

Seam extension for Gemini: it puts the model in the URL PATH, not the body, so parseRequest was widened to (body, path?) and the core passes clientReq.url; Anthropic/OpenAI ignore the arg.

Other honesty rules enforced: audit forwards byte-identical bytes (inject only shapes the assist candidate); createTokenCounter returns null for OpenAI/Gemini (no cheap probe → honest partial, avoiding the illegal-probe-body trap Anthropic's count_tokens had); a price snapshot carries an optional max_prompt_tokens tier ceiling — a Gemini 2.5 Pro prompt >200k prices to NULL, not the wrong base rate; the audit report and kage status break {exact,partial,unavailable}/token/cost down PER PROVIDER and a zero-traffic provider is ABSENT (not a $0 bucket); the proxy emits evidence fail-open after the client is answered, and readAdapterConnection is cached 3s so it never spawns `ps` per request.

Honest boundaries: the proxy is provider-API-scoped (an agent whose provider has no adapter gets no coverage); OpenAI/Gemini exact-COST coverage is lower than Anthropic's (no count-tokens probe; tier ceilings null); per-provider delivery attribution needs a provider column on context_deliveries (not yet added); Azure OpenAI is out of scope.
Evidence: Built as a 4-task workstream (90fd714, 71b689d, 14446bd, c1b3898 + hardening a5d1761/63d07f5/11212c1/7e8c69b), each with an adversarial review that caught a real defect.
Verified by: npm test --prefix mcp (735/735); npm run test:vnext (288/288); dogfood 12/12; each provider's usage semantics source-verified against its live API docs
