# Kage savings has TWO paths now: reuse (assist, primary) + reversible compression (protect, Phase D) — real-body net ~0%

> Kage's savings story has TWO paths as of Phase D this supersedes the earlier "savings is a REUSE thesis, NOT compress…

Kage's savings story has TWO paths as of Phase D (this supersedes the earlier "savings is a REUSE thesis, NOT compression" framing, which was accurate for Phase A/B before a compression path existed). PATH 1 — REUSE (assist mode injection): the proxy INJECTS memory, ADDING tokens to one prompt; the value is the agent avoiding re-reading/re-deriving across a task. This remains the PRIMARY proven value. Reporting still keeps MEASURED (stale_withheld, recalls, caller_answers) distinct from ESTIMATED (tokens_saved heuristic); the end-to-end token delta needs a live A/B and is deliberately not claimed. PATH 2 — REVERSIBLE COMPRESSION (protect mode, new in Phase D): a genuinely reversible (exact original stored in the content-addressed store + a kage-content:<sha256> retrieval marker), byte-preserving-on-failure, measured-not-estimated mechanism wired behind the ProviderGateway seam (Anthropic-only; OpenAI/Gemini pass through). It is REAL and safe but NOT a headline savings source on typical traffic: measured real-body median NET saving is ~0% (git diff ~0.1%, source/JSON passthrough) — compression only pays off on genuinely repetitive payloads (~99% on a repeated-log contrast). The pipeline passes through byte-preserving whenever the compressed output plus marker does not actually shrink the body, so it never claims a saving it did not achieve. kage up keeps AUDIT as the default and lossy_compression defaults false. The live cohort is empty (scripts/vnext-phase-d-report.mjs returns empty_cohort honestly, not a fabricated zero). Reproduce the real-body number with: npm run bench:compression --prefix mcp (benchmarks/compression-realbody-kage.mjs). Net: compression is now a real capability, but the reuse thesis remains the primary value; neither path fabricates an unmeasured saving.
Evidence: Independent verification: build clean; npm test --prefix mcp 1120/1120 + dogfood 12/12; Phase D gate 3/3. Real-body benchmark (npm run bench:compression): real git diff net ~0.1% KEEP, real source passthrough (no compressor supports), real JSON passthrough (net-of-marker does not shrink), synthetic repetitive log ~99% KEEP; real-body median NET saving 0%. scripts/vnext-phase-d-report.mjs --project . --json returns available:true empty:true reason:empty_cohort.
Verified by: npm test --prefix mcp; node --test mcp/dist/vnext/phase-d-gate.test.js; npm run bench:compression --prefix mcp; node scripts/vnext-phase-d-report.mjs --project . --json
