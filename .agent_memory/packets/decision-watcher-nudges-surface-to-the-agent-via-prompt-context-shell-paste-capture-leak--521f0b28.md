# Watcher nudges surface to the agent via prompt-context; shell-paste capture leak closed

> surfacePendingNudges mcp/kernel.ts reads .agent memory/nudges/pending.jsonl, returns unsurfaced nudges as a "⚠ Kage n…

surfacePendingNudges (mcp/kernel.ts) reads .agent_memory/nudges/pending.jsonl, returns unsurfaced nudges as a "⚠ Kage nudges — act on these" block, and rewrites the file marking each surfaced:true so a nudge shows exactly once. The `prompt-context` CLI command (mcp/cli.ts, run by the UserPromptSubmit hook) appends this block — and now emits it even when recall finds no other memory — which is what makes watcher nudges felt by the agent instead of sitting silently in a file. Separately, isSerializedDumpTitle now rejects shell-prompt / terminal-paste titles via /^[\w.+-]+@[\w.+-]+[\s:].*[%$#]\s/ (e.g. "user@host dir % cmd"); isSerializedDumpBody delegates to it, so the guard covers bodies too. That paste class had been re-creating junk capture packets. Both shipped in v2.5.2.
Verified by: npm test (353 unit + 12 dogfood pass) and live `kage prompt-context` through the global 2.5.2 binary surfacing a nudge once, then empty on re-run
