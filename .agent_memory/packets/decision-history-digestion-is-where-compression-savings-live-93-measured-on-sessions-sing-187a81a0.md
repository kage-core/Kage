# History digestion is where compression savings live: ~93% measured on sessions; single-body nets ~0%

> W1+W2 post Phase E goal expansion changed Kage's compression story. W1: the Phase D transform pipeline assist injecti…

W1+W2 (post-Phase-E goal expansion) changed Kage's compression story. W1: the Phase D transform pipeline (assist injection + reversible compression) now binds ALL three providers — providers/openai.ts (chat/completions; prompt_tokens is a FULL total) and providers/gemini.ts (a deterministic LOSSLESS contents<->messages view: {text} parts <-> {type:"text"} blocks, functionCall/functionResponse untouched, serialize restores the exact wire shape) registered in providerAdapterFor; proxy.ts routes the pipeline through the adapter's parse/serialize view; unknown providers stay null (pipeline skipped byte-preserving). W2: the measured truth is that single-body compression nets ~0% on real traffic, so the real mechanism is HISTORY DIGESTION (mcp/vnext/gateway/history.ts): tool payloads OLDER than the live zone are reduced to a deterministic head/error-lines/tail digest + kage-content marker, exact original stored content-addressed FIRST. Measured on a 12-turn session of real repo bodies (npm run bench:compression): final-turn request saving 92.93%, whole-session 93.33%. Load-bearing properties (all tested in history.test.ts): determinism (digested prefix is itself cache-stable across turns — one provider-cache miss, then re-keys), idempotence (guard keys on the digest's own structural line, NOT any "kage-content:" substring — this repo's own diffs contain the prefix and must stay digestible), reversibility (content-addressed dedup: identical payloads across turns share one object + marker), zone safety, fail-open. Policy: history_compression (default FALSE) + history_min_bytes (2048) in budget-policy.ts; requires lossy_compression too. GOTCHA for future work: any idempotence/skip guard over payload text must match the digest's structural format precisely — a substring check on the marker prefix zeroed all savings on this repo because Kage's own sources/diffs mention kage-content:.
Evidence: Suite 1452/1452 aggregate after W1 (21 provider tests) + W2 (8 history tests). bench:compression measured: single-body real-body median net -0.15%; history digestion final-turn 92.93%, session 93.33% on real repo bodies. Commits 5848f0f (W1), f09754b (W2).
Verified by: npm test --prefix mcp; node benchmarks/compression-realbody-kage.mjs; node --test mcp/dist/vnext/gateway/history.test.js + providers/*.test.js
