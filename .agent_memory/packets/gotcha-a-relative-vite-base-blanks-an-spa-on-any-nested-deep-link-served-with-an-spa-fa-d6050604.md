# A relative Vite base blanks an SPA on any nested deep-link served with an SPA fallback

> The knowledge portal platform/web built with vite base:'./' relative . Served by the daemon under /app/ with an SPA f…

The knowledge portal (platform/web) built with vite base:'./' (relative). Served by the daemon under /app/ with an SPA fallback (any unmatched /app/* -> index.html). A ONE-segment deep link like /app/components worked (browser resolves ./assets against /app/). But a TWO-segment deep link like /app/decisions/<slug> resolved ./assets/index.js against /app/decisions/ -> GET /app/decisions/assets/index.js -> the SPA fallback returned index.html (200, text/html) -> the browser executed HTML as JavaScript -> the app never booted -> BLANK #root, and crucially NO clean console error. This silently broke every detail-page bookmark, reload, and shared link. In-app navigation (pushState) hid it because assets were already loaded. Fix: vite base:'/app/' (ABSOLUTE) so assets resolve to /app/assets/... at every route depth; still same-origin so script-src 'self' is satisfied. Diagnosis tip: compare a 1-segment vs 2-segment deep link; check the built index.html asset refs (./assets = relative = bug; /app/assets = absolute = correct); a 200 that returns index.html for an asset path is the tell. Guard: platform/web/src/base-path.test.ts asserts the absolute base.
Evidence: Deep-link /app/decisions/<slug> -> empty #root, hasMain false, no console error; in-app nav to same route rendered fine. Built index.html had src='./assets/index-*.js'. After base:'/app/', src='/app/assets/index-*.js' and the deep link rendered the full detail page. Published 4.0.4 index.html confirmed absolute.
Verified by: platform/web/src/base-path.test.ts (frontend 110/110); browser deep-link render verified

## Verification

Deep-link /app/decisions/<slug> -> empty #root, hasMain false, no console error; in-app nav to same route rendered fine. Built index.html had src='./assets/index-*.js'. After base:'/app/', src='/app/assets/index-*.js' and the deep link rendered the full detail page. Published 4.0.4 index.html confirmed absolute.
