# Kage vNext runtime gate requires canonical three-part Node versions

> Kage vNext local persistence remains isolated behind openVnextDatabase: node:sqlite is referenced only as a type at m…

Kage vNext local persistence remains isolated behind openVnextDatabase: node:sqlite is referenced only as a type at module evaluation and required inside the Node 22.5+-gated open function, preserving legacy Node 18 CLI loading. The runtime gate must accept only canonical three-component Node versions in major.minor.patch form, with no leading zeros except the single component 0, before applying the >=22.5 threshold. Two-part versions and zero-padded forms are malformed and must receive the same helpful runtime error. Migration 001 remains transactional and versioned; evidence events remain append-only by source_fingerprint and receipts idempotent by request_id without estimating unavailable measurements.
Evidence: The regression test first failed because 22.5 was accepted. After requiring exact canonical major.minor.patch components, the focused storage suite passed 10/10 and npm test --prefix mcp passed 444/444 plus 12/12 dogfood.
Verified by: Task 2 specification-review TDD red/green and full test suite

## Verification

The regression test first failed because 22.5 was accepted. After requiring exact canonical major.minor.patch components, the focused storage suite passed 10/10 and npm test --prefix mcp passed 444/444 plus 12/12 dogfood.
