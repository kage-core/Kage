# Kage vNext is collaborative repository intelligence with automatic, cost-controlled agent context

> Kage vNext is defined as a collaborative repository intelligence and agent context platform, not a packet manager, ge…

Kage vNext is defined as a collaborative repository-intelligence and agent-context platform, not a packet manager, generic personal-memory store, or agent runner. The approved direction uses a hybrid local/team architecture: a persistent local daemon records evidence and delivers automatic budgeted context, a canonical repository model compiles features/components/flows/contracts/runbooks/decisions from evidence, and a team workspace handles review, ownership, permissions, metrics, and synchronization. Kage must produce exact per-task value receipts and automatically back off when context increases measured cost. Headroom-inspired reversible context compression belongs in the Context Budget Engine; Ponytail-inspired repository-specific minimalism belongs in the Minimal Change Guard. The program is decomposed, with local runtime and exact measurement as the first implementation phase.
