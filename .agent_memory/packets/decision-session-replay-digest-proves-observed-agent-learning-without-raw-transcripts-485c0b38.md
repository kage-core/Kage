# Session replay digest proves observed agent learning without raw transcripts

> Kage now exposes a privacy preserving session replay digest through kage replay, kage session replay, GET /kage/repla…

Kage now exposes a privacy-preserving session replay digest through kage replay, kage_session_replay, GET /kage/replay, and the viewer dashboard/memory page. It summarizes observed session timelines, paths, commands, durable candidates, and distill commands while setting raw_text_included=false and avoiding raw transcript text in the replay surface. This closes an session-memory replay proof gap while preserving Kage's reviewable memory packet model.
Evidence: Implemented in mcp/kernel.ts, mcp/cli.ts, mcp/index.ts, mcp/daemon.ts, mcp/viewer/app.js, docs/viewer/app.js, README.md, mcp/README.md, docs/guide.html with tests in mcp/kernel.test.ts, mcp/mcp.test.ts, mcp/daemon.test.ts.
Verified by: npm test --prefix mcp -- --test-name-pattern 'session replay digest'; node --check mcp/viewer/app.js; node --check docs/viewer/app.js; CLI smoke: kage observe then kage replay returned raw_text_included=false
