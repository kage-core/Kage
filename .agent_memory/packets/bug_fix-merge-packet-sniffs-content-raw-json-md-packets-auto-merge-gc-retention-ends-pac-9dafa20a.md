# merge-packet sniffs content (raw-JSON .md packets auto-merge); gc retention ends packet immortality

> Rank 7 slices fixed 2026 07 03. MERGE DRIVER: mergePacketFiles mcp/kernel.ts routed sides to a parser by file extensi…

Rank-7 slices fixed 2026-07-03. MERGE DRIVER: mergePacketFiles (mcp/kernel.ts) routed sides to a parser by file extension — .md went to okfConceptToPacket, which fails on the raw-JSON packets that dominate the store, so the driver punted every sync-bot race to manual conflict markers. It now sniffs content: leading "{" tries JSON first, then the OKF parser, then conflict-marker recovery. Proven live the same day: a `git pull --rebase` against a sync-bot commit printed "kage merge-packet: kept theirs side (newest updated_at …)" and completed with zero manual conflicts. GC RETENTION: gcProject skipped deprecated packets forever ("already deprecated"), making dead packets immortal — 32% of the store. Deprecated/superseded packets older than KAGE_GC_RETENTION_DAYS (default 30, env-overridable) are now deleted with the audit tombstone. LIMITATION: retention keys on updated_at, and the sync bot's metadata churn keeps refreshing updated_at on dead packets, so most existing dead weight won't age out until the bot stops rewriting volatile fields into tracked packets (remaining rank-7 work: kage-state sidecar for volatile metadata).
