# Capture guard: ungrounded conversational user utterances routed to pending + withheld from recall

> The serialized dump guard isSerializedDumpTitle/isSerializedDumpBody did not catch raw conversational user utterances…

The serialized-dump guard (isSerializedDumpTitle/isSerializedDumpBody) did not catch raw conversational user utterances captured as packets — e.g. an issue_context with no cited paths whose body is a frustrated chat message. evaluateMemoryAdmission could even score it admissible because keywords like issue/before/why hit the durable-trigger regex. Fix in mcp/kernel.ts: new isUngroundedConversationalCapture(packet) fires ONLY when ALL hold — zero cited paths, no repo grounding signal (no path/file/code-identifier/command, via the shared hasRepoGroundingSignal that observationSignalScore now also uses), AND looksLikeRawUserUtterance (a run of emphatic terminal punctuation or a curated rhetorical second-person-at-the-assistant phrase). The conjunction keeps real ungrounded decisions safe. Wired as defense in depth mirroring the dump guard: evaluateMemoryAdmission subtracts 45 and hard-excludes admit; capture() routes such packets to pending not approved, tags them needs-grounding, and warns instead of hard-deleting; recall withholds them from both approved results and the pending working-memory section.
Evidence: New test in mcp/kernel.test.ts; npm test -> 354 pass + 12 dogfood pass
Verified by: npm test from mcp/
