# kage_context bloat + dump-packet capture: cause and fix

> kage context could return 270k+ chars and overflow the MCP response. Two root causes, both fixed: 1 the index.ts kage…

kage_context could return 270k+ chars and overflow the MCP response. Two root causes, both fixed: (1) the index.ts kage_context handler called queryGraph a second time and emitted a duplicate "## Graph Facts" section on top of recall's "## Related Graph Facts", and neither renderer truncated edge.fact/evidence, so one raw-transcript graph edge (118k chars) was dumped twice. (2) Raw transcript / tool-output / file-content "dump" packets (bodies up to 666KB) entered storage via the auto-distill/observation path because capture()'s dump guard only ran when input.strictCitations was set and only checked the title; their giant bodies became knowledge-graph edge facts.

Fix (mcp/kernel.ts unless noted): added isSerializedDumpBody(body) (true when length > MAX_PACKET_BODY_CHARS=16000, env KAGE_MAX_PACKET_BODY_CHARS, or dump markers / isSerializedDumpTitle on body) and clampInline(text,max) (whitespace-collapse + truncate with marker). queryGraph now filters edges where isSerializedDumpBody(edge.fact) and clamps every rendered field; recall's "## Related Graph Facts" clamps fact+evidence. capture() now rejects dump title OR dump body on EVERY path (removed the strictCitations gate). The distill helpers reusableFileObservation/reusableCommandObservation/reusablePromptObservation skip dumps and clampInline the learning to 1200 chars. gcProject deletes dump packets (isSerializedDumpTitle(title) || isSerializedDumpBody(body)). The index.ts handler dropped the duplicate queryGraph + "## Graph Facts" section and added a 24000-char backstop cap (gains receipt kept outside the cap). Result on the same query: 270,063 -> 6,009 chars, longest line 343. Cleaned 7 legacy dump packets (~1.78MB) from .agent_memory/packets and rebuilt the graph.
Evidence: All 351 mcp tests pass (incl. 3 new: capture dump rejection, gc dump deletion, queryGraph clamp/filter). Measured recall().context_block 270063 -> 6009 chars on the original query; gcProject --dry-run flagged exactly the 7 dump packets.
Verified by: npm test (351 pass) + measured context_block size + gc dry-run
