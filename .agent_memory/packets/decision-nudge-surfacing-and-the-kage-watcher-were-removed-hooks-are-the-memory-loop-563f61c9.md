# Nudge surfacing AND the kage-watcher were removed — hooks are the memory loop

> Both the watcher "nudge" surfacing added 2.5.2/2.5.3, removed in 2.5.4 and the kage watcher background agent itself w…

Both the watcher "nudge" surfacing (added 2.5.2/2.5.3, removed in 2.5.4) and the kage-watcher background agent itself were removed. The kage-watcher agent definition (~/.claude/agents/kage-watcher.md — a user-local Claude Code Agent subagent, never part of the npm package or `kage setup`/`install`, confirmed by grep of kernel.ts/cli.ts/index.ts) is deleted and will not regenerate. Rationale: a persistent polling watcher duplicates what event-driven hooks already do — recall (UserPromptSubmit→prompt-context), verified-memory-at-file-touch (PreToolUse→file-context), and capture (Stop→distill + auto-promote, with the kage-distiller agent for quality). Judgment that genuinely needs an LLM (quality capture, contradiction detection) is better invoked on-demand by a hook than by a parallel loop. DO NOT re-add a persistent watcher; if contradiction-nudges are ever wanted, generate them from a PostToolUse/Stop hook instead. Still current and unchanged: recall, file-context citation injection, capture, the shell-paste capture guard, and the normalizeRepoPath/trailing-slash cleanup. Remaining kage agents: kage-distiller, kage-graph, kage-indexer, kage-memory.
Verified by: ~/.claude/agents/kage-watcher.md deleted; not referenced by kage setup/install (grep clean) so it won't regenerate; no leftover nudge/pending.jsonl code in source; 353 unit + 12 dogfood tests green at v2.5.4
