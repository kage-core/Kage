# Gimmick-removal pass: cut unsubstantiated hype across product and docs, kept real substance

> Ran an audit driven pass to remove hype/decorative elements with no backing evidence, following directly from the pro…

Ran an audit-driven pass to remove hype/decorative elements with no backing evidence, following directly from the professionalism cleanup. 22 fixes across docs/index.html, docs/releases.html, README.md, CONTRIBUTING.md, mcp/kernel.ts, mcp/cli.ts, mcp/kernel.test.ts, plus 2 orphaned asset deletions (docs/kage-stats.svg, docs/assets/kage-og.svg — both already disavowed by an earlier 'Honesty cleanup' commit and referenced nowhere).

Key fixes: docs/index.html — corrected a self-contradicting trust-benchmark stat to 100/100, removed a redundant Star-on-GitHub button (the real live star count already renders via docs/assets/stars.js), reworded 'Kage called it' and 'supercharge' copy, replaced a fabricated '360+ tests' stat with 'Fully tested' (no number), and rewrote the Kage Cloud pricing card from '$0 early access' (implies it ships) to 'Not built yet' with honest gate-disclosure copy matching docs/CLOUD.md's actual waitlist gate (GitHub issue #74, unmet). docs/releases.html — corrected a stale v2.2.1 headline to the actual v3.3.0/July 3 2026 release, dropped a fabricated test count, and removed a false 'dependencies' clause from a '0 accounts or API keys' claim. README.md — removed a stale demo GIF and hero tagline, a deps-0 badge, and a Book-a-demo link with no backing form. CONTRIBUTING.md — replaced the absolute 'zero runtime dependencies' claim (false — three is genuinely used, served client-side via daemon.ts's /vendor/three/ static route for the viewer's 3D graph, not caught by a Node-side import grep) with an honest accounting of the 5 real runtime deps and why each is load-bearing. mcp/kernel.ts and mcp/cli.ts — removed marketing narration from kage demo's console output and the claude-mem audit receipt, removed a decorative progress-bar+divider from kage savings/team output that was redundant with the adjacent exact percentage (applied consistently to both commands despite one judge flagging only one of them), reworded two comments that used undefined 'trust wedge' jargon.

What was explicitly KEPT as legitimate, not gimmick: docs/assets/stars.js (a genuine live GitHub API fetch, verified via curl against api.github.com), the account/OKF badges and comparison table in README.md, the LongMemEval-S 'zero dependencies' line (a true, narrowly-scoped claim about that specific benchmark harness, distinct from the false product-wide claim that was removed), and a load-bearing internal design-rationale comment in kernel.ts that a corrupted judge output had marked ambiguous — independently re-read and reworded for jargon rather than trusted blindly.
Evidence: npm test: 402/402 pass; live preview screenshots of docs/index.html and docs/releases.html; curl verification of github.com/kage-core/Kage star/download counts and issue #74 reaction count
