# A value dashboard must LEAD with what's measured; empty metrics become unlock CTAs, never dead ends

> The portal Overview led with 8 provider cost/trust metric cards net context cost, verified reuse, time to verified ch…

The portal Overview led with 8 provider-cost/trust metric cards (net_context_cost, verified_reuse, time_to_verified_change, understanding_coverage, attach_reliability, runbook_health, ...) that render 'Unavailable' or '0%' on any repo that hasn't run assist-mode proxy traffic + review — while the genuinely MEASURED value (recalls_served, stale_withheld, packets captured, from teamValueReport / TeamReportDto) sat in a section below the fold. Result: the flagship 'here's your value' screen looked broken/empty. The honesty contract (never fabricate; unmeasured = 'Unavailable') is correct and must stay, but LEADING with it is self-sabotage. Fix (4.0.5): OverviewPage now leads with a value hero built from TeamReportDto (recalls served, stale caught, knowledge captured, tokens saved-estimated-labelled), the cost/trust metrics moved BELOW under 'Cost & trust', and each unmeasured metric shows an UNLOCK_HINTS step ('Run kage up --mode assist ...') instead of a dead 'Unavailable'. Also: workspace Team panel renders one honest line when null (not 8 empty cards); branch/commit filled from live git in the daemon (patchGitIdentity in mcp/daemon.ts, out of the pure read-model). Principle for any Kage value surface: lead with measured, label estimates apart, turn an absent measurement into 'how to produce it', never a dead end. The cost/verified-reuse numbers stay Unavailable until assist-mode traffic + review exist — the cold-demo chicken-and-egg; a seeded demo repo with priced receipts is the remaining step for the money-number pitch.
Evidence: Browser: Overview now leads 'Kage served 175 recalls and withheld 559 stale claims across 229 captured memories' with stat tiles; unmeasured metrics show 'Run kage up --mode assist' hints; branch codex/kage-vnext-design commit f9144dc populated. Frontend 110/110, backend 1434/0.
Verified by: platform/web OverviewPage.test.tsx + MetricCard tests (110/110); browser get_page_text of /app/overview
