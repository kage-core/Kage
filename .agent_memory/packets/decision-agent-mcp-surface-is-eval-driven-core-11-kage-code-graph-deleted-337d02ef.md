# Agent MCP surface is eval-driven: core=11, kage_code_graph deleted

> The default agent facing MCP tool surface CORE TOOLS in mcp/index.ts, returned by listTools unless KAGE TOOLS=full/KA…

The default agent-facing MCP tool surface (CORE_TOOLS in mcp/index.ts, returned by listTools() unless KAGE_TOOLS=full/KAGE_ALL_TOOLS=1) is now 11 tools: kage_context, kage_learn, kage_supersede, kage_feedback, kage_pr_check, kage_refresh, kage_skills, plus kage_risk, kage_decisions, kage_dependency_path, kage_docs_search. The last four were promoted from full-only after the agent-trajectory eval (evals/agent-trajectory/) recorded a real headless agent naturally calling each on a plain task. kage_code_graph was DELETED entirely (tool def + callTool handler removed, full registry 67->66) because the eval showed the agent answers "who calls X" with grep and never reaches for it, and kage_context already surfaces caller/dependency-path/risk context (covered by the mcp.test.ts "kage_context includes risk and dependency path" test). Decision rule established: a tool an agent never chooses even on a tailored task does not belong in the agent surface — demote to CLI or delete. mcp.test.ts asserts the exact 11-tool core and that kage_code_graph is absent from the full registry. kernel queryCodeGraph/buildCodeGraph remain (used by kage_context and the CLI `kage code-graph` command).
