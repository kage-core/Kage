# Viewer components must show action next to insight

> Viewer and site components should be judged from the user's action goal: what does the user gain, will they understan…

Viewer and site components should be judged from the user's action goal: what does the user gain, will they understand it, and what should they do next. Metrics, cards, rows, review items, and raw data lists should include a nearby action or decision cue. Avoid throwing raw information at users without explaining the inspection, review, test, capture, or cleanup action it supports.
