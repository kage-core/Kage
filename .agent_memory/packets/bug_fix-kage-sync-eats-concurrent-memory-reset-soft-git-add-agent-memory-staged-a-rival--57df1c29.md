# kage-sync eats concurrent memory: reset --soft + git add .agent_memory/ staged a rival packet as DELETED

> The Kage Sync workflow .github/workflows/kage sync.yml runs on EVERY push to master, so its own push routinely loses …

The Kage Sync workflow (.github/workflows/kage-sync.yml) runs on EVERY push to master, so its own push routinely loses the race to a concurrent commit (a kage learn memory write, a PR merge). Its push-retry reconciliation did: git fetch; git reset --soft origin/master; git add .agent_memory/. Because reset --soft keeps THIS job's older working tree — which never contained the packet the winning commit just added — 'git add .agent_memory/' staged that packet as a DELETION, and the sync commit silently ate freshly-captured memory. Reproduced deterministically (scripts/kage-sync-commit.test.mjs): sync regenerates graph, a concurrent packet lands first, sync's reconcile deletes it. Root cause is NOT staleness/GC/refresh (verified: refresh withholds stale, never deletes; the auto-deprecate step is separately dead — it JSON.parse()s OKF markdown and crashes under continue-on-error). Fix: after reset --soft, 'git checkout origin/<branch> -- .agent_memory/packets/' restores the branch's packets before re-staging, so a concurrent packet is never staged for deletion; the sync's own new packets (absent from the branch) and derived graph/index artifacts still commit. Logic extracted to scripts/kage-sync-commit.sh + regression test wired as npm test:sync. Three real packets were lost this way (vite-base gotcha, packaged-feature gotcha, value-dashboard decision) and recovered from git history via 'git show <delcommit>^:<path>'.
Evidence: Deterministic race repro: buggy reconcile DELETED the concurrent packet; fixed reconcile keeps concurrent + sync-own packets + graph. Test passes with fix, fails without (asserts 'the concurrently-added packet MUST survive'). Lost packets recovered: 8786/8468/10174 bytes.
Verified by: scripts/kage-sync-commit.test.mjs (test:sync) — pass with fix, fail without

## Verification

Deterministic race repro: buggy reconcile DELETED the concurrent packet; fixed reconcile keeps concurrent + sync-own packets + graph. Test passes with fix, fails without (asserts 'the concurrently-added packet MUST survive'). Lost packets recovered: 8786/8468/10174 bytes.
