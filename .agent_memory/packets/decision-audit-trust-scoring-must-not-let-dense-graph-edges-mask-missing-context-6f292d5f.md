# Audit trust scoring must not let dense graph edges mask missing context

> Fact: Kage audit now reports structured engineering memory coverage, code graph precision, memory code evidence, and …

Fact: Kage audit now reports structured engineering-memory coverage, code graph precision, memory-code evidence, and a trust score. Why: Dense memory-code graph edges can make a repository look well-covered even when approved memories lack explicit rationale, verification, or stale conditions. Trigger: When changing audit scoring, knowledge graph edge generation, or memory admission heuristics. Action: Cap memory-code edge scoring by approved packet count and prefer explicit Fact/Why/Action/Verified by/Stale when fields in high-value memories. Verified by: npm test in mcp; node mcp/dist/cli.js audit --project . --json. Risk if forgotten: Future audit changes may overstate repository trust and hide poor structured-memory coverage.
Evidence: Implemented structured context extraction, auditProject, memory-code graph evidence links, and a trust-score cap; tests passed and root audit reported trust_score 80 with structured-memory recommendations.
