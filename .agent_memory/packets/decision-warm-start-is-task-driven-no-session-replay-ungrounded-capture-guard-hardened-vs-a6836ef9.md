# Warm start is task-driven (no session replay); ungrounded-capture guard hardened vs name-drops

> kageResume mcp/kernel.ts no longer renders last session files/commands or a recency ranked " Recent memory" timeline …

kageResume (mcp/kernel.ts) no longer renders last-session files/commands or a recency-ranked "## Recent memory" timeline in its SessionStart context_block. That digest fired BEFORE the task was known (so it could never be targeted) and was the surface that injected raw command dumps and ungrounded junk packets at the top of every session. kageResume now emits only "# Open threads (Kage)" — change-memory, pending drafts to review, reconcile pointers — plus the always-on pinned repo facts (renderPinnedRepoContext). Task-relevant recall happens on the first prompt via prompt-context, and on file read via file-context. The structured report still keeps last_session/recent_memory fields for the API/viewer; they're just not injected. Separately, isUngroundedConversationalCapture was hardened: a new looksFrustratedAtAssistant check (rant-at-the-assistant phrases like "why are you", "don't stop", "it's your job") now OVERRIDES hasRepoGroundingSignal — because the loose grounding check false-positived on platform name-drops (github/pr/x/linkedin) and let frustrated user outbursts through as APPROVED memory (root cause of the recurring "why are you asking me" junk). gcProject now deletes ungrounded-conversational packets on sight, like serialized dumps. CAVEAT: the guard runs in capture/admission (in-process MCP server), so live capture enforces it only after an agent restart (see the capture/distill-needs-rebuild+restart runbook); the CLI path (gc, Stop-hook distill) gets it on global upgrade. Shipped v2.5.6; swept 12 legacy junk packets.
Verified by: 355 unit + 12 dogfood pass; guard now catches the real junk text and 0 false positives on legit notes (tested via dist); 12 ungrounded packets swept; kage resume output confirmed free of Previously/Commands/Recent-memory/junk; published v2.5.6 + global upgraded
