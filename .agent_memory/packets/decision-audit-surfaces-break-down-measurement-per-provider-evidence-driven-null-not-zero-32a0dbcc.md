# Audit surfaces break down measurement per provider (evidence-driven, null not zero)

> Task 4 made the Kage audit surfaces report PER PROVIDER because the proxy is now multi provider anthropic|openai|gemi…

Task 4 made the Kage audit surfaces report PER PROVIDER because the proxy is now multi-provider (anthropic|openai|gemini). `byProvider(receipts)` in mcp/vnext/runtime/commands.ts groups receipts by the receipt's `provider` field and returns a Record keyed by provider, each bucket = { provider, receipts, measurement{exact,partial,unavailable}, token_delta, cost_delta } computed with the SAME shared coverageOf/tokenDelta/costDelta helpers. It is exposed as `by_provider` on VnextStatusReport (kage status) and on scripts/vnext-phase-a-report.mjs. Key honesty decisions: (1) EVIDENCE-DRIVEN/sparse — a provider with no receipts has NO key (absence = no traffic), never a fabricated {0,0,0} or $0; `by_provider` is null (not {}) when the receipt store is unreadable, mirroring `measurement`. (2) The overall total stays ALONGSIDE the split; overall cost_delta still counts only two-sided-priced receipts, so a null-cost/one-sided/tier-ceiling provider is excluded, never summed as $0. (3) Attachment CANNOT be split per provider — context_deliveries rows carry no provider column — so `attachment_by_provider` is always { available:false, reason:"delivery_rows_have_no_provider" }; splitting it needs a delivery-row migration (protocol v1 stays frozen; a delivery is Kage's own record). Gate: mcp/vnext/phase-a-gate.test.ts drives anthropic+openai receipts through the real store and asserts two buckets, gemini absent, openai cost unavailable not zero, overall cost counts only the anthropic two-sided receipt.
Evidence: npm test --prefix mcp green at 734/734 + dogfood 12/12; test:vnext 288/288. Report and status --json on a seeded 2-provider store show anthropic+openai buckets, gemini absent, overall cost_delta.receipts=1 (openai one-sided cost excluded).
Verified by: npm test --prefix mcp (734 pass, 0 fail); node scripts/vnext-phase-a-report.mjs --json on seeded multi-provider store
