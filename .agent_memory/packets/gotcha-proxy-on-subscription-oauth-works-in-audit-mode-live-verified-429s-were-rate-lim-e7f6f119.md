# Proxy on subscription OAuth WORKS in audit mode — live-verified; 429s were rate limits, not a block

> RESOLVES the earlier open gotcha "kage proxy on Claude Code SUBSCRIPTION: base url IS honored, but the OAuth path ret…

RESOLVES the earlier open gotcha "kage proxy on Claude Code SUBSCRIPTION: base-url IS honored, but the OAuth path returns 429 (diagnosis open)". First live authenticated run through the vNext audit-mode proxy on a real Claude Code SUBSCRIPTION (OAuth, not API key), 2026-07-16:

- ANTHROPIC_BASE_URL is honored and authenticated subscription traffic flows through the proxy SUCCESSFULLY. Three real requests completed with full usage reported (133,314 total input tokens / 987 output across them), producing the first three real transformation receipts and three truthful audit "skipped" deliveries.
- The 429s that motivated the original gotcha appear to be GENUINE rate limiting, not an OAuth-path block: two 429 rate_limit_error responses were interleaved WITH successful requests in the same session (Claude Code retries them automatically). Audit mode forwards byte-identical bytes, so no mutation could have caused them.
- One transient 401 "OAuth access token has been revoked" appeared on the first authenticated attempt — Claude Code refreshed its token and continued. Expect and ignore a single such 401 at session start through a proxy.
- Behavior notes for reading the proxy log: only non-2xx responses get per-request log lines; successes appear only in the stats counters. "0 memories injected" is CORRECT in audit mode (it composes and measures but never injects — the skip-deliveries record that truthfully). Receipts appear only for requests where recall produced a candidate.
- Measurement quality of a default audit run is PARTIAL (the forwarded side is measured by provider usage; the unsent candidate is unmeasured without --count-tokens), and token/cost deltas are honestly unavailable — exactly as designed, not a defect.

What remains genuinely unverified on the OAuth path: assist mode (body mutation) — the original 429-on-mutation hypothesis is untested either way; audit's byte-identical forwarding sidesteps it. Test assist separately before claiming it works on subscription.
Evidence: Proxy stats after the live session: 7 req, 3 captured, 3 receipts, 133,314 input / 987 output tokens — three authenticated requests succeeded with full provider usage. kage status shows 3 anthropic receipts (partial coverage, honest nulls for deltas) and 3 truthful audit skip-deliveries. Interleaved failures: one 401 "OAuth access token has been revoked" (transient, Claude Code refreshed and continued) and two 429 rate_limit_error (retried; successes in between prove no hard block).
Verified by: Live run 2026-07-16: kage proxy --mode audit --port 8788 fronting api.anthropic.com, real Claude Code subscription session pointed at it via ANTHROPIC_BASE_URL
