# Privacy <private> tags are redacted by stripPrivateSpans before any memory write

> Convention: all memory write paths must redact <private ...</private spans before persisting. The single shared sanit…

Convention: all memory write paths must redact <private>...</private> spans before persisting. The single shared sanitizer is stripPrivateSpans() in mcp/kernel.ts (exported for tests), backed by PRIVATE_SPAN_PATTERN — case-insensitive, multiline, and an unclosed <private> redacts to end-of-string. Hook points: capture() sanitizes title/summary/body/context at entry (before scanSensitiveText, so secrets inside private tags are stripped rather than blocking the write), learn() sanitizes learning/title/evidence/verifiedBy before deriving titles, and observe() sanitizes text/summary/command before hashing and writing observation records. Distill is covered transitively: it reads already-sanitized observations and writes through capture/learn. Any new write path that persists free text must call stripPrivateSpans first.
Verified by: node --test --test-name-pattern "private" dist/kernel.test.js (7 pass) and full suite 155/155
