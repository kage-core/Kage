# vNext /v2/context: kernel-sourced trust, byte-capped inputs, exact section accounting

> Phase A Task 4 composes automatic context through a replaceable ContextSource commits 5aef382, 236985b, b8a3f54 . Thr…

Phase A Task 4 composes automatic context through a replaceable ContextSource (commits 5aef382, 236985b, b8a3f54). Three constraints are non-obvious, and each was a real defect in the first implementation that review caught.

1. Packet trust MUST come from the kernel's exported packetVerificationLabel (mcp/kernel.ts:9761), which reads quality.stale then freshness.verification plus freshness.last_verified_at. The first implementation invented its own check against packet.quality.verified and quality.verification_status — fields the kernel never writes anywhere — so the "verified" trust tier was unreachable and every packet came back "approved". That read was also unguarded, while every other kernel site defensively writes `packet.quality ?? {}`. A packet with no quality key therefore threw a TypeError that the handler laundered into 503 context_source_unavailable, meaning one malformed packet on disk would take /v2/context down for every query while misreporting a code bug as an availability problem. Route trust through the LegacyKernelFunctions seam; never reimplement the kernel's verification rule.

2. estimated_tokens must be charged in exact serialized BYTES, accumulated and rounded once at the end — not ceil() per section. A section costs its own JSON plus the single comma joining it to the previous section; the enclosing brackets are envelope structure, not section cost (charging brackets per-section would make an empty capsule report a nonzero budget). Per-section rounding overshot token_budget by one token on exact alignment boundaries. Separately, the capsule echoes request.query verbatim, so query, repo_id, and task_id must be byte-capped at validation and REJECTED rather than truncated — otherwise a large query blows the stated budget by orders of magnitude while the capsule still reports estimated_tokens: 0. The guarantee actually enforced is estimateTokens(JSON.stringify(capsule)) <= token_budget + MAX_CAPSULE_ENVELOPE_TOKENS.

3. LegacyContextSource.find is async in name only. recall, kageRisk, and kageTeammateBrief are synchronous kernel calls, and kageRisk falls back to a full code-graph build on a cold repo. That blocks the runtime's single event loop, so no in-process timeout can preempt it — a timeout wrapper would return early while the loop stays blocked, which is a dishonest fix and was deliberately not added. Bounding the inputs (MAX_CONTEXT_QUERY_BYTES, MAX_CONTEXT_IDENTIFIER_BYTES, MAX_CONTEXT_PATHS, MAX_CONTEXT_PATH_BYTES) bounds the abuse case, not the cold-build case: a fully legal request on a cold repo can still exceed Task 5's 500 ms abort budget. Fixing that means moving kernel work off the request thread, which is a change of shape rather than a hardening tweak.

Structural rule: LegacyContextSource is the ONLY vNext module permitted to import mcp/kernel.ts. No adapter or gateway may import the kernel.
Evidence: Two independent reviews of 5aef382 found these three defects; hardening landed in 236985b and b8a3f54. The boundary-accounting regression test was mutation-checked: it fails under the old per-section rounding and passes under byte accumulation.
