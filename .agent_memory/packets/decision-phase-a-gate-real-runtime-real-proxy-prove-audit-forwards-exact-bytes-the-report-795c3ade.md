# Phase A gate: real runtime + real proxy prove audit forwards exact bytes; the report must never fake a zero

> mcp/vnext/phase a gate.test.ts is the Phase A gate and is deliberately end to end: it starts the real local runtime r…

mcp/vnext/phase-a-gate.test.ts is the Phase A gate and is deliberately end-to-end: it starts the real local runtime (real loopback HTTP, real bearer token, real SQLite, the shipped WorkerContextSource), posts a handshake + prompt event through the shipped adapter client, requests a real capsule, writes a context_deliveries row into the real store, and runs a real `startProxy(mode: "audit", countTokens: true)` against a fake PROVIDER (the only stand-in, so CI never bills Anthropic). It asserts agent_request_unchanged === true, receipts.length === 1, receipts[0].mode === "audit", daemon_outage_result === "failed_open" (the runtime is actually closed and the shipped hook re-run), and mcp_calls_required === 0 (proved by mcp/dist/index.js never appearing in require.cache, so no tool call was possible). The gate was verified to BITE: flipping the proxy to assist mode makes agent_request_unchanged false. scripts/vnext-phase-a-report.mjs is the audit artifact: an empty or unreadable period reports NULL for every measurable value (never 0, which would read as "Kage ran all week and cost nothing"); measurement_scope is explicitly "transformed_requests" because receipts exist only for actually-transformed requests; prompt_mutations is a MEASURED count of receipts whose transformed body was forwarded (0 in audit, and a test proves it counts 1 in assist); and the report never conflates an exact TOKEN delta with an exact COST delta — audit receipts carry provider_input_cost_after_usd null because count_tokens gives a token total with no cache breakdown, so cost_delta is "unavailable", never before - 0.
Evidence: npm run test:vnext --prefix mcp: 196/196 pass. npm test --prefix mcp: 637/637 + dogfood 12/12. Report on this repo prints available:false, reason:no_receipt_store, every metric null.
