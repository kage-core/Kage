# Daemon REST exposes complete agent memory operations

> After comparing HTTP-capable agent workflows, Kage daemon REST now exposes the same repo local memory primitives HTTP…

After comparing external memory tool's HTTP-agent story, Kage daemon REST now exposes the same repo-local memory primitives HTTP-only agents need: /kage/context for combined recall, graph facts, validation, risk, and dependency context; /kage/capture and /kage/learn for durable packet writes; /kage/feedback for helpful/wrong/stale signals; plus /kage/setup-doctor for setup readiness. These endpoints reuse Kage's existing packets, graphs, recall, audit, and validation instead of creating a separate REST memory store.
Evidence: external memory tool advertises broad REST support for non-MCP agents. Kage previously documented Aider REST mode with only recall, observe, and distill. The daemon now advertises and routes context, capture, learn, feedback, setup-doctor, recall, observe, distill, and reports.
