# Repo professionalism cleanup: removed internal strategy docs, fixed stale metadata, closed the public-docs feature gap

> A full 6 way parallel audit root docs, benchmark/ vs benchmarks/, CLI command surface, docs/ coherence, package.json/…

A full 6-way parallel audit (root docs, benchmark/ vs benchmarks/, CLI command surface, docs/ coherence, package.json/CI, mcp/ source tree) found and fixed real issues, not cosmetic ones: (1) OKF_PIVOT.md (40KB internal strategy memo, 'lead strategist' voice, competitive/red-team analysis) and launch/GROWTH_PLAN_2026-07.md (internal go-to-market doc) were sitting in the repo, unlinked from anywhere — removed via git rm, confirmed via user choice, still recoverable from git history. (2) Root server.json AND mcp/server.json were BOTH stale (versions 2.3.3 and 2.3.0 vs actual 3.3.0) and inconsistent with each other; mcp/server.json was never referenced by any script/workflow and not shipped in the npm package (files whitelist excludes it) — consolidated to one canonical root server.json at the correct version with the real custom domain (kage-core.com, confirmed via docs/CNAME) as websiteUrl. (3) docs/viewer/ was a confirmed stale mirror of mcp/viewer/, missing the dollar-estimate-removal cleanup and the entire Team-link sidebar feature — synced console.js and index.html (left docs/viewer/data/ and demo/ untouched, they're the real CI-regenerated hosted-demo content). (4) CHANGELOG.md had a real formatting bug: a stray '## 2.0.1' heading was concatenated onto the end of the v1.0.0 bullet with no line break, and its content was stranded at the very bottom of a newest-first file — fixed the line break and moved the entry to its correct chronological position between v2.0.2 and v2.0.0. (5) mcp/package.json was missing repository/homepage/bugs/author fields (real npm discoverability gap) — added, verified via npm pack --dry-run. (6) 9 of mcp/cli.ts's FULL_USAGE commands (metrics, profile, xray, capabilities, decisions, module-health, graph-insights, audit, quality) had zero usage description, making an 11-command 'report on this repo' cluster indistinguishable from the help text alone — added precise descriptions, reusing this session's own verified test titles as the wording so they're already fact-checked, not guessed. capabilities/capability-audit/readiness was found to already be a clean single-canonical-name-plus-silent-aliases pattern, not actual redundancy — no change needed. (7) benchmark/ (singular, Python SWE-bench ablation) and benchmarks/ (plural, JS retrieval/scale suite) are BOTH live and intentionally coexist for different benchmark categories, not dead/duplicate — added a one-line cross-reference at the top of each README so a visitor immediately understands the distinction instead of assuming duplication. (8) ROADMAP.md never mentioned OKF despite that being the entire current product positioning, and its Now/Next sections referenced already-shipped work as still-planned — rewrote to reflect OKF as the core bet and today's real Next items (design partner over hardening, public docs catching up). (9) The public site was silent about kage proxy/cloud/team/savings — all real, tested, shipped features — and worse, index.html's pricing section told visitors 'Team memory: coming soon, book a demo' when a self-hosted, tested version has existed since earlier this session. Added a full 'Team memory & distribution' section to guide.html's CLI-by-job reference and corrected the pricing copy to say self-hosting is free and available today, with 'Team (managed)' now specifically scoped to Kage running it for you.
Evidence: npm test: 402/402 pass. npm pack --dry-run: 20 files, 339.4kB, clean (cloud-client.js/cloud-server.js correctly included, no server.json/test files leaked). Live-verified via preview: guide.html's 4 new method cards render correctly styled; index.html's corrected pricing card renders with no overflow. 9 stale packets reverified with cleanup-pass evidence; the pre-existing OKF_PIVOT.md citation packet was already deprecated (no action needed).
Verified by: npm test: 402/402 pass; npm pack --dry-run clean; live preview verification of guide.html and index.html
